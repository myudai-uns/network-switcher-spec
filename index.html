要件定義書

かんたんネット切替ツール

Windows 端末の IPv4 設定（固定IP ⇔ 自動取得）をボタン操作で切り替えるツール

版数：v1.0（ドラフト） 作成日：2026-06-22 対象OS：Windows 10 / 11 形態：単一exe ＋ 設定ファイル

1. 背景・目的

1.1 背景（現状の課題）

現場端末では、ネットワーク設定を Windows の「アダプターの設定 → インターネットプロトコル v4 → プロパティ」画面から手入力で切り替えている。

  • IPアドレス／サブネットマスク／デフォルトゲートウェイ／優先・代替DNS の固定値と自動取得（DHCP）を、場面に応じて切り替える必要がある
  • 手順が長く（10ステップ以上）、毎回4つの数字を手打ちするため、打ち間違い・設定漏れの事故が起きやすい
  • 端末ごとに固定IPの末尾が異なるため、作業が属人的になりやすい

1.2 目的

上記の手入力作業をボタン操作に置き換える。設定される項目・値は現状の手入力と完全に同一とし、「結果は変えず、手順と事故だけ無くす」ことをゴールとする。

1.3 非ゴール（やらないこと）

  • ネットワーク構成・IP体系そのものの変更（現状をそのまま移管するだけ）
  • IPv6設定、プロキシ設定、VPN設定
  • サーバー連携・集中管理（各端末単体で完結）

2. 対象・前提

項目内容
対象OSWindows 10 / 11
対象アダプタWi-Fi（主）／有線LAN（可能なら）の両方。実行時に選択
実行権限管理者権限（起動時にUAC昇格を自動要求）
配布形態単一exe（NetworkSwitcher.exe）＋設定ファイル profiles.json
利用者現場端末の操作者（ITに詳しくない人を想定）
技術方式設定変更は Windows 標準コマンド netsh 経由（手入力と同一結果）

3. 機能要件

ID機能内容
FR-1アダプタ選択物理アダプタ（Wi-Fi・有線）を一覧表示。接続中を上位表示。「更新」で再取得。
FR-2現在設定の表示選択中アダプタの 取得方法／IP／マスク／GW／DNS を表示。切替後は自動再取得。
FR-3固定IP切替
（番号入力方式）		 共通テンプレート 10.208.{セグメント}.{ホスト番号} で固定IPを生成。
  • セグメント選択：10.208.38.x / 10.208.40.x
  • ホスト番号入力：末尾のみ入力（38系=1〜243／40系=1〜190）
  • 範囲外・数字以外はエラーで適用不可
  • マスク・GW・優先/代替DNS はセグメントごとに自動付与
  • 適用前にプレビュー表示
FR-4端末ごとの
設定保持		 入力したセグメント・番号を端末内（last_input.json）に保存し、次回起動時に自動復元。端末ごとに一度入力すれば以後は入力不要。exeは全端末共通、端末差は保存ファイルで吸収。
FR-5自動取得(DHCP)切替IPアドレス・DNS の両方を自動取得（DHCP）に戻す。
FR-6設定ファイル管理セグメント定義・レンジ・マスク・GW・DNS を profiles.json で管理。再読込可。不在時はサンプル自動生成。記述エラーは日本語提示。
FR-7入力チェック適用前に形式・範囲を検証。不正時は中止し原因を日本語表示。

4. 非機能要件

区分要件
操作性不慣れな人でも「選ぶ→押す」の2アクションで完了。日本語UI・大きめボタン。
正確性netsh 経由で設定。手入力と同一の結果を保証。
安全性範囲外IPを物理的に入力不可とし、誤設定を防止。
応答性取得・適用は別スレッドで実行しUIを固めない。
可搬性単一exeで動作。Python等のインストール不要。
保守性IP体系の変更は profiles.json 編集のみで対応（再ビルド不要）。
権限管理者権限が無い場合は明確に案内。

6. 確認シート（現場で記入してください）

本ツールを「現状の手入力運用」と完全に同一にするため、現状の実値が必要です。
現場端末を1台開き、設定 → ネットワーク → アダプターの設定変更 → Wi-Fiを右クリック → プロパティ → インターネットプロトコル v4 → プロパティ の画面に表示されている値を、下表に記入してください。

記入のコツ：固定IPが設定されている端末を見れば、マスク・ゲートウェイ・DNSが表示されています。その数字をそのまま転記すればOKです。
#確認項目現状（仮の値）確定値（記入欄）
Q1サブネットマスク255.255.255.0
Q2ゲートウェイ（38系）10.208.38.254
Q3ゲートウェイ（40系）10.208.40.254
Q4優先DNS（38系 / 40系）GWと同じ
Q5代替DNS（あれば）なし
Q6有線LANも対象に含めるか含める
Q7同一端末で38系/40系の両方を使うか両方選択可
Q8ホスト番号レンジに変更はあるか
（現状：38系=1〜243 / 40系=1〜190）		変更なし
確定後：上記 Q1〜Q8 が埋まれば、profiles.json に反映するだけで要件が確定し、運用開始できます。記入後の表を画面ごと共有いただくか、値をそのままお知らせください。

※ 入力内容はこのページ内だけに保持され、サーバーには送信されません（印刷・PDF用）。

7. 「現状の移管」であることの確認

本ツールは新しい仕組みを導入するものではなく、今まで人が手で開いていた設定画面と全く同じ項目・同じ値を、Windows標準コマンドで正確に書き込むものです。
確認ポイント答え
設定される項目IP・サブネット・GW・優先/代替DNS。現状の手入力項目と完全一致
設定される値profiles.json の値。現状の手入力の数字を入れれば結果は同一
Wi-Fi / 有線現状同様、対象を選択可能
自動取得に戻す現状同様に戻せる

移管を確信するための検証手順（推奨）

  1. 現状の端末を1台、設定画面で開き、4つの数字をメモ（IP・マスク・GW・DNS）
  2. その数字を profiles.json に反映
  3. テスト端末1台で「アプリで切替 → 設定画面を開いて、手入力時と同じ数字になっているか目視確認」
  4. 一致を確認できたら全端末へ展開
